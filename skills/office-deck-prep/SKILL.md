Use this workflow when the user asks to inspect a deck, prepare speaker notes, export slides, or start a presentation in Keynote or PowerPoint.

Use read-only deck metadata and slide listing first. Add or revise speaker notes only with bounded app-specific tools, and start presentation mode only when the user has clearly requested it. Do not use arbitrary AppleScript, Office macros, VBA, shell commands, or unrestricted screen control.

For presentation starts, exports, and speaker-note edits, name the deck target and slide or output path. Expect approval for presentation control, exports, or edits involving sensitive or low-integrity source material.
